Renderjs

A WebGL rendering framework for JavaScript

Why build Yet Another Rendering Framework?

There are plenty of great frameworks out there that can target the web. My favourites are Unity, BabylonJS, and three.js.

But as far as I know, none of them shares the following three properties:

  • Specifically targeted at the web, using modern Javascript concepts and patterns
  • Declarative in nature, so that you only need to write controller code in the most extreme cases
  • Flexible in the sense of being general-purpose enough to build a simple site animation, a full-blown game, or a 3D editor using the same library.

However, the more interesting reason is actually just because I can. You'll notice a number of pieces of code in the source that could have been outsourced to well-established libraries. This isn't a case of not-invented-here so much as an exercise in the pure joy of developing something.

Renderjs is built for the love of 3D coding.
